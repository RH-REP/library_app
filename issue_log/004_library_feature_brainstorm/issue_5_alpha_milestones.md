# Issue #5 Alpha Development Milestones

## 依頼

ユーザーは、先のブレインストーミング表から、まず開発すべき最小構成を取り出し、アルファ開発でどこまでできればよいかをマイルストーンとして定義するよう依頼した。

## 判断

- 今回は候補を広げる段階ではなく、アルファ完了条件を切る段階である。
- `goal.md` の必須条件に直結する要素だけを残す。
- 特に「AI なしで検索できる」「原本へ戻れる」「Web で参照できる」を中核に置く。

## アルファで必須としたもの

- 原本の保存場所
- 原本のマスターテーブル
- 最小メタデータ
  - タイトル
  - 著者
  - 種別
  - タグ
  - 状態
  - 原本 ID
- 非 AI 検索
- 一覧/詳細の Web/HTML 画面
- 基本登録導線

## アルファから外したもの

- フォルダ階層
- 用語集
- PDF scan program
- OCR
- 推薦
- 保存済み検索条件
- 読書ログ
- 高度なメモ運用

## マイルストーン

- M1: データ境界と原本契約
- M2: 登録フロー
- M3: 非 AI 検索
- M4: 一覧/詳細/原本参照 UI
- M5: アルファ検証

## 完了条件

- 原本を分離保存できる。
- 原本マスターテーブルで管理できる。
- 最小メタデータ付きで登録できる。
- AI なしで検索できる。
- 一覧/詳細から原本へ戻れる。
- PDF を代表的な原本として扱える。

## 次の候補 issue

- 原本マスターテーブルの first cut schema を作る。
- 登録フローの入出力を決める。
- 検索 UI の最小画面を切り出す。
- タグ運用ルールの最小案を作る。
