Alpha Source Sample / HTML

要求工学・仕様化の最小メモ

何を作るかを明確にするための代表的な考え方を、alpha 開発向けの 簡易サンプルとしてまとめた HTML 原本。

目的: 開発前に「誰が」「何を達成したいか」を定義し、後続の検索、 登録、参照機能の優先順位を判断できる状態を作る。

代表的な手法

  • 要件定義: 目標、制約、非機能条件を明文化する。
  • ユースケース: 利用者が行う操作の流れを記述する。
  • ユーザーストーリー: 小さな価値単位に要望を分解する。
  • 契約による設計: 入力、出力、前提条件を明確にする。

alpha で確認したいこと

Input

登録対象は PDF、HTML、画像などの原本を含むか。

Search

AI なしで検索するために最低限必要なメタデータは何か。

Traceability

検索結果から原本へ戻る導線をどの識別子でつなぐか。
